## Maintenance Rules

- Keep active docs inside `docs/`, not in the repository root.
- Use the root [README.md](../README.md) only as the project entry point.
- Treat `docs/plans/` as historical context, not as setup or operational source of truth.
- Remove temporary outputs, one-off audit exports, and stale duplicate docs instead of letting them accumulate.
