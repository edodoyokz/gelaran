# Product Requirements Summary

This file is a cleaned summary of the early project requirement notes so the project keeps one readable product-scope reference.

## Objective

Build a local ticketing management and on-ground ticketing platform for events in Solo/Surakarta, with the option to scale to broader city-event usage later.

## Example Event Types

- cultural performances
- sports matches
- seminars and workshops
- music gigs
- party and nightlife events

## Planned Product Updates

- FAQ page for customer self-service
- customer-support follow-up flow when FAQ is not enough
- regional tax calculator
- commission-fee calculator
- seating-plan flow for ticket selection and presentation demos

## Main Roles

- Admin
- Organizer
- Customer
- Event-day operator roles such as gate staff or POS staff

## Expected Benefits

- locally owned ticketing workflow
- clearer reporting for local event revenue
- less manual operational work through digital processes

## Notes

- Current implementation details belong in code, tests, and active docs.
