# Plans and Design Notes

This directory stores historical task plans, design writeups, implementation notes, and completion summaries produced during project work.

## What This Folder Is For

- design records for a specific task
- implementation plans tied to a date or ticket
- completion summaries and cleanup notes

## What This Folder Is Not For

- local setup instructions
- operational runbooks
- current product requirements
- root project onboarding

Use these docs as background context only. For active documentation, start from [docs/README.md](../README.md).

## Naming Convention

Most files follow one of these patterns:

- `YYYY-MM-DD-<topic>-design.md`
- `YYYY-MM-DD-<topic>.md`
- `YYYY-MM-DD-<ticket-or-workstream>-completion-summary.md`

## Guidance

- Keep new plans in this folder if they are task-specific.
- Do not duplicate setup or runbook content here.
- When a plan becomes obsolete but still useful as history, keep it here instead of the repository root.
